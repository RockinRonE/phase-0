All About Version Control

Version Control and You

Sept., 10th 2015

Why Use Version Control?

  • Client: We would like a login form for our customers. Can you do that?
  • You: Of course! I'll work on that. (Works for hours, updates site).
  • Client: Nice login, but we actually decided against it. Can you remove it?
  • You: Oh...sure. (Deletes code )
  • Client, few days later: Actually we would like it! My boss thought it was great.
  • You: Argh! I have to code it from scratch again!

In the world of web development this situation can happen. Version control is the best way to protect yourself while allowing you to do so much more. Version control allows you to take your original code, known as a master, and duplicate it as a "branch." A branch is your copy of the code for you to do whatever you'd like to it. Your changes are tracked and you save, or "commit" them. When you're finished you then merge this new code with the master. Why is this beneficial? Say the above situation actually happened. The programmer removes the login but then the client wants it back. All the programmer would have to do is restore the previous version!

Additional Benefits

  1. Collaboration

    2. Version control allows every team member to have their own copy of the master code to make changes to. The updated code then can be compared to the original and if everyone approves, be merged into the master copy. This is a great way for a team to work on multiple facets of a project simultaneously. One of the most popular services is GitHub, which I'll discuss below.

  2. What's Going On?

    3. Version control keeps track of changes that are made so you will always know exactly what changed, who did it, and when. This is useful in case a bug was introduced into the code unnoticed. It can be tracked

  3. Backup

    4. Since your teammates all have a copy of the master code, version control becomes a way of backing up the code.

GitHub

GitHub uses the version control system Git to keep track of changes on its site, hence the name GitHub. GitHub is a remote repository in which you can store your code. This is great for teams because each person can download a copy of the master code on GitHub to their own computer. That person then can branch off of it, makes changes, and then send it back to GitHub for their peers to review. This is known as a "pull request." A pull request gives you the opportunity to review the changes made by your teammate and read their reasoning behind it. If it looks good you can then accept the changes and merge them into the master code.
